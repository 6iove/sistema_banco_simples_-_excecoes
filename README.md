# Projeto – Sistema Bancário com Tratamento de Exceções

Este projeto foi desenvolvido como **trabalho da disciplina Ferramentas de Linguagem de Programação**, do curso de Análise e Desenvolvimento de Sistemas.

O objetivo da atividade é **refatorar e melhorar um código previamente criado na disciplina de Paradigma Orientado a Objetos**, no semestre anterior.
A proposta consiste em:

* Revisar o código original.
* Organizar melhor a estrutura das classes.
* Implementar validações.
* Adicionar tratamento de exceções personalizado.
* Melhorar a robustez e legibilidade geral do projeto.

O sistema simula operações bancárias básicas, como cadastro de clientes, abertura de contas, transações financeiras e verificações simples de dados (como email, CPF e senha).

## Tecnologias e conceitos utilizados

* Python 3
* Programação Orientada a Objetos (POO)
* Modularização
* Exceções e exceções personalizadas
* Validações de entrada (email, CPF, senha, etc.)
* Boas práticas de organização de código

## Objetivo educacional

O foco principal é demonstrar domínio dos seguintes tópicos:

1. **Aplicação de POO na prática**
2. **Uso correto de exceções**
3. **Melhorias de design e qualidade do código**
4. **Aplicação de padrões básicos de validação e segurança**

## Status do projeto

Em desenvolvimento e em constante aprimoramento conforme os requisitos da disciplina.
